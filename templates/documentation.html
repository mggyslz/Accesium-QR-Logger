System Overview

Accesium is a modern QR-based access control system designed to track and manage entry/exit of personnel in facilities, schools, offices, or any secure location.

Key Features

Secure Authentication

Two-factor authentication with email verification and trusted device management

Real-time Analytics

Live dashboards with activity charts, heatmaps, and attendance tracking

User Management

Complete user lifecycle management with roles and access rules

QR Code Scanning

Fast, contactless entry with unique QR codes and PIN verification

Dark Mode

Comfortable viewing experience with automatic theme switching

Multiple Locations

Support for multiple entry points with location-based tracking

System Components

Component Purpose Users
Admin Portal System management, user creation, analytics Administrators
User Portal Personal dashboard, QR code access, attendance records All registered users

Getting Started

First-Time Setup

Note: The first admin account must be created before any other accounts can be added.

Creating the First Admin Account

  1. Navigate to the landing page at /
  2. Click on "Admin Login"
  3. Click "Sign Up" (only visible if no admin exists)
  4. Fill in the registration form:
    • Username (minimum 3 characters)
    • Email address
    • Full name
    • Password (minimum 6 characters)
  5. Click "Create Admin Account"
  6. You'll be automatically logged in to the admin dashboard

Administrator Guide

Dashboard Overview

The admin dashboard provides a comprehensive view of system activity:

  • Currently Inside: Real-time count of people in the facility
  • Total Users: Number of registered users
  • Active Location: Current scanning location
  • Recent Activity: Latest entry/exit events
  • Activity Charts: 7-day and hourly activity graphs

User Management

Creating New Users

  1. Navigate to Registered Users section
  2. Fill in the "Add New User" form:
    • Email: User's email address (required for login)
    • Full Name: User's complete name
    • Role: Select from Staff, Employee, Student, Teacher, or Visitor
  3. Click "Create User"
  4. The system will:
    • Generate a username automatically (firstname.lastname format)
    • Create a temporary password
    • Generate a 6-digit PIN for QR scanning
    • Create a unique QR code
    • Send credentials via email
Automatic Credentials: Users receive their username, password, and PIN via email. They must change their password on first login for security.

Managing Existing Users

  • Search: Use the search box to find users by name or role
  • View QR Code: Click on the QR code image to view full size
  • Download QR Code: Click the download link to save the QR code
  • Delete User: Click the "Delete" button to permanently remove a user

Attendance Records

Viewing Attendance

  1. Navigate to Attendance Records
  2. Use filters to refine results:
    • Search: Filter by name
    • Date Range: Select from/to dates
    • Action: Filter by IN or OUT only
    • Location: Filter by specific location
  3. Browse paginated results (50 records per page)

Exporting Data

Two export formats are available:

  • Excel Export: Includes multiple sheets (All Records, IN Records, OUT Records, Summary) with formatting
  • CSV Export: Simple comma-separated values for compatibility

Access Rules

Control when and where users can access the facility.

Creating Access Rules

  1. Navigate to Access Rules
  2. Choose user selection method:
    • Bulk Selection: Select multiple users using checkboxes
    • Single Selection: Use dropdown for individual user
  3. Configure rule parameters:
    • Rule Type: Whitelist (only allow) or Blacklist (deny)
    • Location: Specific location or all locations
    • Time Range: Allowed hours (optional)
    • Date Range: Valid date period (optional)
    • Specific Dates: Comma-separated dates (optional)
  4. Click "Add Rule(s)"
Rule Priority: Blacklist rules take precedence over whitelist rules. If a user matches both, access will be denied.

Location Management

Set the active scanning location:

  1. Use the Active Location dropdown in the top bar
  2. Select from predefined locations:
    • Main Gate
    • Reception
    • Office Area
    • Classroom
    • Library
    • Supply Area
    • Back Gate
  3. All scans will be logged with this location

Security Settings

Session Management

  • Session Timeout: Configure auto-logout time (15-480 minutes)
  • Warning Period: Users get a 2-minute warning before timeout

Login Security

  • 2FA Requirement: Enable/disable mandatory two-factor authentication
  • Account Lockout: Enable lockout after failed attempts
  • Lockout Threshold: Set number of attempts (3-10)
  • Lockout Duration: 15 minutes by default

Changing Admin Password

  1. Navigate to Settings
  2. Find the Change Password section
  3. Enter:
    • Current password
    • New password (minimum 6 characters)
    • Confirm new password
  4. Click "Change Password"
  5. You'll receive an email notification of the change

User Guide

First Login

  1. Check your email for credentials from the admin
  2. Navigate to User Login
  3. Enter your username or email and temporary password
  4. If 2FA is enabled:
    • Enter the 6-digit code sent to your email
    • Optionally check "Remember this device" to skip 2FA for 30 days
  5. You'll be prompted to change your password
Password Change Required: For security, you must change your temporary password on first login.

QR Scanner Guide

Accessing the Scanner

  1. Navigate to /scanner
  2. Allow camera permissions when prompted
  3. Scanner interface will display with live video feed

Scanning Process

For Entry (Last Action: OUT or First Scan)

  1. User presents their QR code to camera
  2. System identifies the user
  3. System checks access rules for the current location
  4. If allowed, system prompts for PIN entry
  5. User enters their 6-digit PIN
  6. System logs IN with timestamp and location
  7. Success message displays with user name

For Exit (Last Action: IN)

  1. User presents their QR code to camera
  2. System identifies the user
  3. System automatically logs OUT (no PIN required)
  4. Success message displays with user name
Automatic Logic: The system automatically determines if a scan is for entry or exit based on the user's last action. Exit scans don't require PIN for faster processing.

Scanner Display

Right Side: Information Panel

Security Features

Authentication

Two-Factor Authentication (2FA)

Enhanced security layer requiring email verification:

  • Email Codes: 6-digit verification codes sent to registered email
  • Time-Limited: Codes expire after 10 minutes
  • One-Time Use: Each code can only be used once
  • Trusted Devices: Option to skip 2FA for 30 days on trusted devices

Password Security

  • Hashing: All passwords are securely hashed using industry-standard algorithms
  • Salt: Unique salt for each password prevents rainbow table attacks
  • Force Change: Admin-created accounts require password change on first login
  • Strength Requirements: Minimum 6 characters with letters and numbers/symbols

Account Lockout

Protection against brute-force attacks:

  • Failed Attempts: Configurable threshold (default: 5 attempts)
  • Lockout Duration: 15 minutes automatic lockout
  • Email Notification: User notified when account is locked
  • Auto-Reset: Counter resets on successful login

Session Management

Session Security

  • CSRF Protection: All forms protected with CSRF tokens
  • Session Regeneration: Session ID regenerated on login/logout
  • Secure Cookies: HttpOnly and Secure flags enabled
  • Inactivity Timeout: Configurable auto-logout (default: 30 minutes)

Timeout Warning

  1. System monitors user activity
  2. At 2 minutes before timeout, warning modal appears
  3. User can:
    • Extend Session: Continue working
    • Logout: End session immediately
  4. If no action taken, automatic logout occurs

Data Protection

Input Validation

All user inputs are validated and sanitized:

  • Username: Alphanumeric, underscore, hyphen only (3-50 chars)
  • Email: Valid email format required
  • Name: Letters, spaces, hyphens, apostrophes only (2-100 chars)
  • XSS Prevention: HTML tags and scripts stripped from inputs

Rate Limiting

Prevents abuse and DOS attacks:

Endpoint Limit Window
Login attempts 10 requests 60 seconds
User registration 5 requests 300 seconds
Password change 5 requests 300 seconds
Export data 5-7 requests 300-700 seconds

Access Control

Role-Based Access

  • Admin: Full system access, user management, settings
  • User: Personal dashboard, QR code, own attendance records
  • Scanner: QR scanning only (no authentication required)

Access Rules

Granular control over facility access:

  • Location-Based: Allow/deny specific locations
  • Time-Based: Restrict access to specific hours
  • Date-Based: Set valid date ranges or specific dates
  • Priority: Blacklist rules override whitelist rules

Audit & Logging

Activity Logging

All system actions are logged:

  • Access Logs: Every entry/exit with timestamp and location
  • Login Attempts: Success and failure tracking with IP addresses
  • Suspicious Activity: Invalid inputs, failed validations, rate limit hits
  • User Changes: Account creation, modification, deletion

Notifications

Real-time alerts for security events:

  • Failed Login: User notified of failed attempts
  • Account Locked: Email sent when account is locked
  • Password Changed: Confirmation email on password change
  • New Device Login: Alert when logging in from new device

Troubleshooting

Login Issues

Problem: "Invalid username/email or password"

Solutions:

  • Verify username/email spelling (case-sensitive)
  • Check password carefully (case-sensitive)
  • Ensure Caps Lock is off
  • Contact admin if you're a new user waiting for activation

Problem: "Account temporarily locked"

Solutions:

  • Wait 15 minutes for automatic unlock
  • Check email for lockout notification
  • Contact admin if you didn't attempt multiple logins

Problem: "Invalid or expired verification code"

Solutions:

  • Check if code is less than 10 minutes old
  • Enter all 6 digits exactly as shown
  • Click "Resend Code" to get a new one
  • Check spam/junk folder for email
  • Verify email address is correct in your profile

Scanner Issues

Problem: Camera not working

Solutions:

  • Check browser permissions for camera access
  • Try a different browser (Chrome recommended)
  • Ensure camera is not in use by another application
  • Refresh the page and allow camera again
  • Check if camera hardware is functioning

Problem: QR code not scanning

Solutions:

  • Hold QR code steady for 2-3 seconds
  • Ensure adequate lighting on QR code
  • Clean the QR code (no smudges or damage)
  • Position QR code 6-12 inches from camera
  • Keep QR code flat and parallel to camera
  • Try printing a new QR code if damaged

Problem: "Access denied" message

Solutions:

  • Check if there's an access rule blocking entry
  • Verify current time is within allowed hours
  • Confirm correct location is set in system
  • Contact admin to review access rules

Email Issues

Problem: Not receiving emails

Solutions:

  • Check spam/junk folder
  • Add system email to safe senders list
  • Verify email address in profile is correct
  • Check if email server is configured (admin)
  • Try a different email address

Problem: 2FA emails delayed

Solutions:

  • Wait up to 5 minutes for email delivery
  • Click "Resend Code" if needed
  • Check internet connection
  • Contact admin if emails consistently delayed

Dashboard Issues

Problem: Statistics not updating

Solutions:

  • Refresh the page (Ctrl+R or Cmd+R)
  • Clear browser cache
  • Check internet connection
  • Try a different browser

Problem: Charts not displaying

Solutions:

  • Ensure JavaScript is enabled
  • Disable browser extensions temporarily
  • Clear browser cache and cookies
  • Update browser to latest version

Data Export Issues

Problem: Export not downloading

Solutions:

  • Check browser download settings
  • Disable popup blocker temporarily
  • Try a different browser
  • Reduce date range if exporting large dataset
  • Check available disk space

Problem: Excel file won't open

Solutions:

  • Ensure you have Excel or compatible software
  • Download file again (may be corrupted)
  • Try opening with Google Sheets or LibreOffice
  • Use CSV export instead if Excel unavailable

Frequently Asked Questions

General Questions

Q: What browsers are supported?

A: Accesium works best on modern browsers including:

  • Google Chrome (recommended)
  • Mozilla Firefox
  • Microsoft Edge
  • Safari (iOS/macOS)

For QR scanning, Chrome provides the best camera support.

Q: Can I use Accesium on mobile devices?

A: Not yet. Mobile support is planned for a future release.

Q: Is my data secure?

A: Yes. Accesium implements industry-standard security measures including password hashing, CSRF protection, rate limiting, and secure session management. See the Security Features section for details.

Q: What happens if I forget my password?

A: Contact your administrator at qrcodecode49@gmail.com to reset your password. See the detailed instructions in the User Questions section below.

Admin Questions

Q: How many admins can the system have?

A: Multiple admins can be created. The first admin creates their account through signup, and additional admins can be added by existing admins.

Q: Can I customize the predefined locations?

A: Currently, locations are predefined in the system configuration. Contact your developer to modify the location list.

Q: How do I reactivate a deleted user?

A: Deleted users are permanently removed. You'll need to create a new account with a new QR code.

Q: Can I export attendance data for a specific user?

A: Yes, use the search filter in Attendance Records to find the user, then export the filtered results.

User Questions

Q: Why do I need to change my password on first login?

A: This is a security measure. Temporary passwords sent by email should be changed immediately to ensure only you know your password.

Q: Can I have multiple QR codes?

A: No, each user has one unique QR code. If your QR code is lost or damaged, contact admin for a replacement.

Q: What if I forget my password?

A: Contact your administrator to reset your password:

  1. Send an email to: qrcodecode49@gmail.com
  2. Use this format:
    Subject: Password Reset Request - [Your Name] Good day, I need to reset my password for the Accesium system. Username/Email: [Your login username or email] Full Name: [Your complete name] Reason: Forgot password Thank you. Best regards, [Your Name]
  3. Wait for the administrator to generate a new temporary password
  4. Check your registered email for the new credentials
  5. Login and change your password immediately
Security Note: For security purposes, the administrator can only send temporary passwords to your registered email address. Make sure to change it immediately upon login.

Q: What if I forget my PIN?

A: Your PIN is displayed in your user dashboard under "My QR Code". You can always log in to retrieve it.

Q: How long do trusted devices stay trusted?

A: Trusted devices remain active for 30 days from the date added. After that, you'll need to complete 2FA again.

Scanner Questions

Q: Why doesn't exit require a PIN?

A: For efficiency and faster processing, exit scans only require QR code verification. The system knows you're exiting based on your last action being IN.

Q: What if someone scans twice accidentally?

A: The system tracks the last action. If you're already IN and scan again, it will log you OUT. Contact admin if there's an error in the logs.

Q: Can the scanner work offline?

A: No, the scanner requires an internet connection to verify users and log access in real-time.

Technical Questions

Q: Where is data stored?

A: All data is stored in a SQLite database with proper encryption and security measures.

Q: Are QR codes encrypted?

A: QR codes contain encoded tokens that are validated against the database. The actual data is not sensitive but the tokens are unique and secure.

Q: What timezone does the system use?

A: All timestamps are stored in UTC and converted to Philippine Time (Asia/Manila) for display.

Best Practices

For Administrators

  • Regular Backups: Export attendance data regularly (weekly/monthly)
  • User Audits: Periodically review user list and remove inactive accounts
  • Access Rules: Keep access rules simple and well-documented
  • Security Settings: Enable 2FA and account lockout for enhanced security
  • Password Policy: Remind users to use strong, unique passwords
  • Location Updates: Always set correct location before scanning sessions
  • Monitor Activity: Check recent logs daily for anomalies
  • Email Configuration: Ensure SMTP is properly configured for notifications

For Users

  • Password Security: Never share your password or PIN
  • QR Code Care: Keep QR code clean and undamaged
  • Trusted Devices: Only mark personal devices as trusted
  • Session Timeout: Always logout when using shared computers
  • Profile Updates: Keep email address current for notifications
  • Regular Checks: Review your attendance records periodically
  • Report Issues: Notify admin immediately of scanning problems

For Scanner Operators

  • Camera Maintenance: Clean camera lens regularly
  • Lighting: Ensure adequate lighting at scanning location
  • Monitor Display: Watch for error messages and anomalies
  • User Assistance: Help users position QR codes correctly
  • Location Verification: Confirm correct location is set
  • Browser Updates: Keep browser updated for best performance
  • Connection: Ensure stable internet connection
